## üí° Key Concepts Explained

### Constraint Types
- **Strict**: Subject MUST be at specific day/period
- **Forbidden**: Subject CANNOT be at specific day/period
- Format: Numeric days (1-5 Mon-Fri), periods (1-7)
- Storage: PostgreSQL SubjectConstraint table

### System Components
- **Database**: SubjectConstraint table with indexed queries
- **API**: 3 endpoints for CRUD operations
- **Algorithm**: Updated to accept and apply constraints
- **Frontend**: Constraint management UI card

### Data Flow
Database ‚Üí Backend API ‚Üí Algorithm ‚Üí Timetable with constraints
